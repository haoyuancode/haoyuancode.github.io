基于hexo在windows搭建博客步骤

2021年8月30日 晚上

此文章内容与实际操作有出入，待更新后在做参考!!!

前提

使用hexo搭建博客前提条件:

安装并配置好Git(用户名和邮箱)

安装nodejs

  1. 下载并安装nodejs

    https://nodejs.org/en/

安装cnpm

  1. win打开命令操作符分别输入 nodejs -v npm -v 查看对应的版本，如果正常显示版本号证明nodejs安装成功

  2. 安装cnpm(由于访问外网网速慢，此处使用淘宝镜像版)

    npm install -g cnpm –registry=https://registry.npm.taobao.org

    安装后使用cnpm -v 查看版本是否安装成功

安装hexo

cnpm install -g hexo-cli

安装后使用 hexo -v 查看版本是否安装成功

创建博客目录并初始化

  1. mkdir blog(可任意设置存放博客文件名)

如果有错误可以删除创建的此目录直接重头再来

此路径在用户目录下

  1. 进入博客目录并初始化博客

    cd blog 进入博客目录

    hexo init 初始化博客(此处网络慢建议使用手机热点)

  2. 进入blog目录查看是否生成基础框架内容

    查看是否有source 、themes、_config.yml等文件

启动并自定义博客

  1. hexo s 启动博客

    启动成功后默认地址是使用localhost:4000进行访问

  2. hexo n “文件名”

    创建新的博客文章

    创建的文章路径在 source/_posts

  3. 编写文章内容

    可使用typora编写，格式为markdown

  4. hexo g

    生成博客内容

    注意要退出到 blog路径下

  5. hexo s

    再次启动博客，使用localhost:4000访问查看

    使用技巧：

    创建新的博客后使用localhost:4000预览，然后在推送到远端

将博客部署到远端(GitHub)

  1. 创建新的仓库

    仓库名必须和GitHub用户名相同

    例如：sixsixsix.github.io

    此地址将是以后在浏览器访问的地址

  2. 安装Git部署插件

    命令提示符进入blog路径

    cnpm install –save hexo-deployer-git

  3. 设置_config.yml

    可以用记事本打开_config.yml

    在 #deployment下 设置

    type: “git”

    repo: (github中新建仓库的地址)

    branch: master

    !!!注意冒号后一定要加空格

  4. hexo d

    将博客部署到远端

    此处将输入两个命令：

    • 第一次输入自己的GitHub账户名

    • 第二次输入GitHub密码

      如果弹出有GitHub窗口，输入GitHub密码即可

  5. 推送成功

    推送成功后访问GitHub新建仓库，会发现多了博客生成的内容

    使用仓库名进行测试访问

部署成功

部署成功后使用新建的GitHub仓库名进行访问
